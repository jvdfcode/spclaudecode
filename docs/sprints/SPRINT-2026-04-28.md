## Critério de sprint review (GO/NO-GO de produção)

- [ ] Login com user real funciona: cadastra → confirma email → login → redirect `/dashboard`

- [ ] Lead capturado e persistido em tabela `leads` (evidência: `SELECT count(*) FROM leads` > 0 no Supabase dashboard)
- [ ] `curl -sI https://smartpreco.app/api/cron/cleanup-ml-cache` sem Authorization retorna `HTTP 401`
- [ ] Sentry recebeu ao menos 1 evento de teste visível no dashboard (`smoke-test-prod`)
- [ ] CI verde em push direto a main: `gh run list --branch main --limit 3` mostra todos `completed / success`
- [ ] `curl -sI https://smartpreco.app/privacidade` retorna `HTTP 200` com conteúdo LGPD
- [ ] **Eventos de funil presentes**: `SELECT count(*) FROM funnel_events WHERE event_name IN ('calculo_iniciado', 'resultado_exibido', 'cta_clicado', 'email_submetido') > 0`

**Resultado possível:**
- **GO** — todos os itens `[x]` com evidência → sprint fechado, produção ativa
- **NO-GO** — 1 ou mais itens sem evidência → listar pendências, decidir: resolver hoje ou carryover para SPRINT seguinte
