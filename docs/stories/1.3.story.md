## User Story

Como desenvolvedor,  
quero CI/CD configurado e a aplicação deployada na Vercel,  
para que cada PR seja validado automaticamente e o deploy em produção aconteça sem intervenção manual.

---

## Contexto Técnico

Configura o pipeline de qualidade e o deploy automático. O GitHub Actions roda lint + typecheck + test + build em cada PR. A Vercel detecta o repositório e faz deploy automático a cada merge na `main`, além de preview deployments por PR.

**Repositório GitHub:** `jvdfcode/spclaudecode`  
**Deploy target:** Vercel (frontend + API Routes)  
**CI:** GitHub Actions (ubuntu-latest, Node 20, pnpm 9)

## ci.yml Completo

```yaml
name: CI

on:
  push:
    branches: [main, 'feature/**']
  pull_request:
    branches: [main]

jobs:
  ci:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup pnpm
        uses: pnpm/action-setup@v3
        with:
          version: 9

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: pnpm

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Lint
        run: pnpm lint

      - name: Type check
        run: pnpm typecheck

      - name: Test
        run: pnpm test --coverage

      - name: Build
        run: pnpm build
        env:
          NEXT_PUBLIC_SUPABASE_URL: ${{ secrets.NEXT_PUBLIC_SUPABASE_URL }}
          NEXT_PUBLIC_SUPABASE_ANON_KEY: ${{ secrets.NEXT_PUBLIC_SUPABASE_ANON_KEY }}
```
