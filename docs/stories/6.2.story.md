## User Story
Como vendedor, quero uma experiência fluida, rápida e sem fricção em todo o app, para que usar o SmartPreço seja mais simples do que usar uma planilha.

## Acceptance Criteria
- [ ] AC1: Tempo de carregamento inicial < 2s (Vercel + Next.js SSR medido no Vercel Analytics ou Lighthouse)
- [ ] AC2: Feedback visual em todas as ações (loading spinners, success toasts, error states)
- [ ] AC3: Formulários salvam estado automaticamente — não perdem dados ao navegar entre tabs
- [ ] AC4: Responsividade validada em: desktop 1440px, laptop 1280px, tablet 768px
- [ ] AC5: Mensagens de erro amigáveis em todo o app (sem stack traces expostos ao usuário)
- [ ] AC6: Onboarding: tooltip ou tour guiado na primeira sessão do usuário
- [ ] AC7: Favicon, metadata OpenGraph e título correto em todas as páginas

## Escopo

**IN (incluído nesta story):**
- Metadata, favicon e OpenGraph em todas as páginas
- ErrorBoundary e páginas de erro customizadas
- Auditoria e correção de feedback visual (loading/toast/error)
- Verificação de persistência de estado nos formulários
- Correções de responsividade nas 3 resoluções alvo
- Tour de onboarding na primeira sessão
- Auditoria de performance (Lighthouse)

**OUT (fora desta story):**
- Redesenho de UX ou novos fluxos funcionais
- Internacionalização (i18n)
- Dark mode
- Rate limiting nas API Routes (pós-MVP)
- Monitoramento de erros com Sentry (pós-MVP)

- Verificar se há `'use client'` desnecessário que força componentes ao cliente
- Lighthouse score alvo: Performance ≥ 90 (CI pode verificar com `@lhci/cli`)

### AC2 — Feedback Visual Universal
[Source: architecture/architecture.md#3-estrutura-de-pastas]
- shadcn/ui já fornece `Skeleton`, `Spinner`, `Toast` — usar consistentemente
- Mapeamento de pontos de loading a auditar:

### AC3 — Persistência de Estado nos Formulários
[Source: architecture/architecture.md#ADR-02]
- Os cálculos vivem no browser (React state via hooks) — ao navegar entre tabs do FlowTabs o estado já deve ser preservado pois é estado compartilhado
- Verificar que `usePricingCalculator`, `useScenarioSimulator` e `useMercadoSearch` estão no nível correto (pai das tabs) para não perder estado
- Para formulário da Calculadora: garantir que `CostForm.tsx` usa estado controlado no hook pai, não estado local
- Opcional: `localStorage` para persistir rascunho se o usuário fechar a aba

### AC4 — Responsividade
[Source: architecture/architecture.md#1-visao-geral — Desktop 1280px+ / Tablet 768px+]
- Tailwind breakpoints: `md:` = 768px, `lg:` = 1024px, `xl:` = 1280px, `2xl:` = 1536px
- Pontos críticos a validar:
  - `ScenarioTable.tsx` — tabela ampla, scroll horizontal em tablet
  - `MarketSummaryPanel.tsx` — gráfico de distribuição em telas menores
  - `SkuGrid.tsx` — grid de cards (2 colunas em tablet, 3+ em desktop)
  - `AppShell.tsx` / `Sidebar.tsx` — sidebar colapsável em tablet
- Testar com DevTools: 768px, 1280px, 1440px

### AC5 — Error Boundaries e Mensagens Amigáveis
[Source: architecture/architecture.md#11-checklist-de-segurança]
- Criar `src/components/ui/ErrorBoundary.tsx` (React Error Boundary)

- Página 404 e 500 customizadas em `src/app/not-found.tsx` e `src/app/error.tsx`

### AC6 — Onboarding
[Source: docs/prd/03-ui-design.md — contexto de UX]
- Solução leve: detectar primeira sessão via `localStorage.getItem('onboarded')`
- Tour guiado simples com tooltips sequenciais (biblioteca leve: `intro.js` ou implementação manual com popover do shadcn/ui)
- Pontos do tour: 1) Calculadora de Custo → 2) Simulador → 3) Bloco Mercado → 4) Decisão
- Botão "Pular tour" sempre visível
- Ao concluir: `localStorage.setItem('onboarded', 'true')`

### AC7 — Metadata e SEO
[Source: architecture/architecture.md#8-variaveis-de-ambiente — NEXT_PUBLIC_APP_URL]
- Next.js App Router: usar `export const metadata` em cada `layout.tsx` e `page.tsx`
- Favicon: `src/app/favicon.ico` (e variantes para Apple Touch Icon)
- OpenGraph padrão no `src/app/layout.tsx`:
  ```typescript
  export const metadata: Metadata = {
    title: 'SmartPreço — Precificação inteligente para vendedores ML',
    description: 'Calcule custos reais, simule cenários e compare com o mercado.',
    openGraph: { title: '...', description: '...', url: process.env.NEXT_PUBLIC_APP_URL }
  }
  ```
- Cada página sobrescreve título: `export const metadata = { title: 'Calculadora | SmartPreço' }`

### Issues Identificadas

**[MEDIUM] `role="dialog"` inadequado no WelcomeTour**
- Arquivo: `src/components/onboarding/WelcomeTour.tsx:64`
- O componente é inline (não modal overlay) — `role="dialog"` implica um widget modal com trap de foco (`aria-modal="true"`)
- Sem trap de foco implementado, screen readers podem ter comportamento inesperado
- Fix sugerido: substituir por `<section aria-label="Tour de onboarding do SmartPreço">` ou adicionar `aria-modal="false"`

**[MEDIUM] OpenGraph sem URL canônica**
- Arquivo: `src/app/layout.tsx:13`
- A story especificava `url: process.env.NEXT_PUBLIC_APP_URL` no openGraph, mas não foi incluído
- Impacto: SEO e social sharing sem URL canônica definida
- Fix sugerido: adicionar `url: process.env.NEXT_PUBLIC_APP_URL` ao objeto openGraph

**[LOW] ErrorBoundary sem `componentDidCatch` para logging**
- Arquivo: `src/components/ui/ErrorBoundary.tsx`
- `getDerivedStateFromError` captura o erro mas não há logging (ex: `console.error` controlado)
- Aceitável para MVP (Sentry está explicitamente fora do escopo), mas dificulta debug em produção
- Fix sugerido: adicionar `componentDidCatch(error, info) { console.error('[ErrorBoundary]', error.digest ?? error.message) }`
