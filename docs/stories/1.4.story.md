# Story 1.4 — Estrutura de Navegação e Layout Base

**Epic:** EPIC-01 — Fundação & Infraestrutura  
**Status:** Draft  
**Agente:** @dev  
**Esforço estimado:** M  
**Dependências:** Story 1.2 concluída

---

## User Story

Como vendedor,  
quero uma navegação clara com as seções principais do app,  
para que eu consiga me localizar e acessar qualquer módulo rapidamente sem me perder.

---

## Contexto Técnico

Implementa o layout principal do app autenticado: shell com sidebar (desktop) ou bottom nav (mobile), header com nome do usuário e logout, e as páginas placeholder de cada módulo. Esta story entrega o esqueleto navegável do produto — sem lógica de negócio ainda.

**Paleta de cores (conforme PRD § 3):**
- Azul escuro como cor primária
- Verde para estados positivos/viáveis
- Vermelho para estados negativos/prejuízo
- Amarelo para estados de atenção

**Rotas a criar:**
- `/dashboard` — página inicial com boas-vindas
- `/calculadora` — placeholder (implementado no EPIC-02)
- `/skus` — placeholder (implementado no EPIC-04)
