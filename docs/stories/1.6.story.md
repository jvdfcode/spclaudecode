## User Story

Como motor de cálculo e módulo de ML API,  
quero uma configuração central com todas as regras e constantes do Mercado Livre,  
para que nenhuma regra de negócio do ML esteja hard-coded espalhada pelo código.

---

## Contexto Técnico

O arquivo `src/lib/mercadolivre.config.ts` é o **único lugar** onde as regras do Mercado Livre residem no código TypeScript. Qualquer outro arquivo que precise de uma constante ML (taxa, threshold, rate limit, keyword de filtro) **importa daqui** — nunca declara a constante localmente.

**Por que centralizar:**
- Quando o ML mudar uma taxa (acontece com frequência), só há um lugar para atualizar
- O motor de cálculo e o módulo ML API compartilham as mesmas constantes sem duplicação
- Facilita testes: mockar `mercadolivre.config.ts` cobre todos os consumidores de uma vez

**Fontes das regras:**
- Taxas: `docs/architecture/schema.md` § 6 (seed data com fontes ML oficiais)
- API: `docs/epics/EPIC-05-EXECUTION.yaml` (tech_notes)
- Posicionamento: EPIC-05 (P25/P75 para below/aligned/above)
- Status de viabilidade: EPIC-02 (≥20% viável, 10-19% atenção, <10% não viável)

## Acceptance Criteria

- [x] **AC1:** Arquivo `src/lib/mercadolivre.config.ts` criado com as seções:
  - `ML_API` — base URL, site ID (`MLB`), rate limit em ms
  - `ML_FEES` — objeto com taxas de comissão por tipo de anúncio (1 parcela)
  - `ML_INSTALLMENT_FEES` — objeto nested `[listingType][installments]` com % de parcelamento
  - `ML_FILTERS` — array de palavras-chave para filtrar kits/combos
  - `ML_POSITIONING` — thresholds de posicionamento (P25, P75)
  - `ML_CACHE` — TTL do cache em ms e em horas
  - `VIABILITY_THRESHOLDS` — thresholds de classificação por margem
  - `STATUS_LABELS` — mapeamento de status para texto em português

  - `free`: 0%

- [x] **AC4:** Array de filtros de kits com ao menos 12 palavras-chave:
  `['kit', 'combo', 'par de', '2x', '3x', '4x', '5x', '10x', 'pacote', 'caixa com', 'conjunto', 'lote']`
